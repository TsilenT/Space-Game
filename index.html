Space Game

A public design project combining the route pressure and ship management of Faster Than Light with the persistent crew, research, and lethal tactical missions of the original X-COM.

FTL decides where disaster happens. X-COM decides who survives it.

Strategic Layer

Choose jumps, manage fuel and hull, repair systems, research alien tech, and keep ahead of a spreading galactic threat.

Tactical Layer

Board enemy ships, defend your own corridors, rescue civilians, retrieve artifacts, and survive turn-based firefights under fog of war.

MVP Target

  • One player ship
  • One enemy ship
  • Four persistent crew
  • Three enemies
  • Salvage, wounds, and repairs

Your coding agent can inspect the machine, prepare the workspace, run tests, create branches, commit, push, and open a pull request. It should only pause when it needs a login, permission, or creative decision from you.

Kickoff prompt · Codex or Claude Code

Take ownership of onboarding me to the Space Game repository at https://github.com/TsilenT/Space-Game.

Handle the technical setup yourself: open or clone the repository, inspect the environment, prepare any tools the project actually needs, and use GitHub for me. Do not give me a list of shell or Git commands to run unless you are blocked by a login or permission that requires me.

Read AGENTS.md, README.md, and SPEC.md before changing anything. Then interview me about the creative and gameplay decisions that are still open. Ask a small batch of focused questions at a time and explain important tradeoffs in plain language.

After the interview, record the decisions in the repository and propose the smallest playable vertical slice. Do not begin a large implementation until I approve the plan.
Working agreement: you choose what the game should feel like. The agent handles implementation mechanics and reports what it changed, what it verified, and what still needs a human decision. Additional guidance lives in AGENTS.md.
